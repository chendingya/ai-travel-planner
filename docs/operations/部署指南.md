## Docker Compose 部署

### 1. 使用根目录 docker-compose.yml（单服务）

项目已内置：

```yaml
services:
  app:
    build:
      context: .
      args:
        NODE_BASE: ${NODE_BASE:-node:18-bookworm-slim}
        NPM_REGISTRY: ${NPM_REGISTRY:-https://registry.npmjs.org}
        PIPER_MODEL_URL: ${PIPER_MODEL_URL-}
        PIPER_MODEL_CONFIG_URL: ${PIPER_MODEL_CONFIG_URL-}
    image: ai-travel-planner:latest
    ports:
      - "3002:3002"
    env_file:
      - backend/.env
    environment:
      TTS_PROVIDER: ${TTS_PROVIDER:-piper}
      TTS_PIPER_MODEL: ${TTS_PIPER_MODEL:-/app/runtime/piper/model.onnx}
      TTS_PIPER_CONFIG: ${TTS_PIPER_CONFIG:-/app/runtime/piper/model.onnx.json}
      TTS_PIPER_MODEL_URL: ${TTS_PIPER_MODEL_URL:-}
      TTS_PIPER_CONFIG_URL: ${TTS_PIPER_CONFIG_URL:-}
```

    说明：

    - `env_file: backend/.env`：把变量注入容器运行时环境。
    - `docker compose --env-file backend/.env ...`：让 Compose 在解析 `${...}`（如 build args）时也读取同一份 `.env`。

### 2. 启动服务

```bash
# 构建并启动（Compose V2）
docker compose --env-file backend/.env up -d

# 查看状态
docker compose ps

# 查看日志
docker compose logs -f app
```

### 3. 停止服务

```bash
# 停止所有服务
docker compose down

# 停止并删除卷
docker compose down -v
```

### 4. 更新服务

```bash
# 重新构建镜像
docker compose --env-file backend/.env up -d --build

# 重启服务
docker compose --env-file backend/.env restart app
```

## 云平台部署

### ModelScope（使用 Node 镜像）

#### 1. 原理
- 容器需监听 `0.0.0.0:7860`，平台将该端口映射到外网。
- 本项目后端读取环境变量 `PORT` / `HOST`，无须改代码即可切换端口。

#### 2. 文件准备
- 保持根目录 Dockerfile（多阶段构建，前端产物复制到 `/app/public`，后端提供静态与 API）。
- 推送完整代码到 ModelScope 空间仓库。

#### 3. 平台环境变量
- 必填：`PORT=7860`、`HOST=0.0.0.0`
- Supabase：`SUPABASE_URL`、`SUPABASE_SERVICE_ROLE_KEY`、`SUPABASE_ANON_KEY`
- AI 配置：`AI_TEXT_PROVIDERS_JSON`、`AI_IMAGE_PROVIDERS_JSON`
- 地图：`AMAP_KEY`、`AMAP_SECURITY_CODE`、`AMAP_REST_KEY`
- MCP：`MCP_12306_URL` / `MCP_12306_TOKEN`、`MCP_AMAP_URL` / `MCP_AMAP_AUTHORIZATION`、`MCP_BING_URL` / `MCP_BING_TOKEN`
- TTS（Piper，可选）：`TTS_PROVIDER=piper`、`TTS_PIPER_MODEL=/app/runtime/piper/model.onnx`、`TTS_PIPER_CONFIG=/app/runtime/piper/model.onnx.json`，可配 `TTS_PIPER_MODEL_URL` / `TTS_PIPER_CONFIG_URL` 在容器启动时自动下载模型；若访问 huggingface 受限，可加 `TTS_PIPER_MIRROR_HOST=hf-mirror.com`

#### 4. 构建与访问
- 触发空间构建，等待完成。
- 平台会提供外网 URL，直接访问即可。

#### 5. 常见问题
- 端口不通：未设置 `PORT=7860` 或 `HOST=0.0.0.0`。
- 前端空白：后端未提供静态目录；本项目已内置 `/app/public` 静态服务与 SPA 兜底路由。
- env-file 解析错误：Docker 的 `--env-file` 不支持多行 JSON，请将 JSON 压成单行或使用 `-e` 注入。

### 配置最佳实践

1. **安全第一**
   - 使用 `.env` 文件存储敏感信息
   - 不要提交 `.env` 到版本控制
   - 使用强密码和 API 密钥

2. **环境隔离**
   - 开发环境：`.env.development`
   - 测试环境：`.env.test`
   - 生产环境：`.env.production`

3. **配置验证**
   - 启动前验证所有必需的配置项
   - 使用默认值和配置模板
   - 提供清晰的错误提示

4. **在线配置管理**
   - 可通过 `GET/PUT /api/provider-config` 管理当前登录用户的 provider 配置
   - 配置按 `user_id` 维度持久化在 Supabase `ai_provider_configs` 表
   - 需要在后端配置 `PROVIDER_CONFIG_ENCRYPTION_KEY` 才能保存（API Key 加密入库）
   - 保存成功后无需重启服务；新配置会在当前用户后续请求中立即生效，且不会影响其他用户

## 数据库初始化

### Supabase 配置

#### 1. 创建 Supabase 项目

1. 访问 [Supabase](https://supabase.com/)
2. 注册并创建新项目
3. 记录项目 URL 和 Keys

#### 2. 执行初始化脚本

在 Supabase 控制台的 SQL Editor 中执行 `supabase-setup.sql`：

```sql
-- 用户表
CREATE TABLE IF NOT EXISTS users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  email TEXT UNIQUE NOT NULL,
  display_name TEXT
);

-- 旅行计划表
CREATE TABLE IF NOT EXISTS travel_plans (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  destination TEXT NOT NULL,
  days INTEGER NOT NULL,
  budget NUMERIC,
  people INTEGER,
  preferences TEXT,
  itinerary JSONB,
  budget_breakdown JSONB,
  tips JSONB
);

-- AI 聊天会话表
CREATE TABLE IF NOT EXISTS chat_sessions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  title TEXT,
  messages JSONB,
  meta JSONB
);

-- 创建索引
CREATE INDEX idx_travel_plans_user_id ON travel_plans(user_id);
CREATE INDEX idx_chat_sessions_user_id ON chat_sessions(user_id);

-- 启用行级安全
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE travel_plans ENABLE ROW LEVEL SECURITY;
ALTER TABLE chat_sessions ENABLE ROW LEVEL SECURITY;

-- 创建 RLS 策略
CREATE POLICY "Users can view their own data" ON users
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Users can insert their own data" ON users
  FOR INSERT WITH CHECK (auth.uid() = id);

CREATE POLICY "Users can view their own travel plans" ON travel_plans
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Users can create their own travel plans" ON travel_plans
  FOR INSERT WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update their own travel plans" ON travel_plans
  FOR UPDATE USING (auth.uid() = user_id);

CREATE POLICY "Users can delete their own travel plans" ON travel_plans
  FOR DELETE USING (auth.uid() = user_id);

CREATE POLICY "Users can view their own chat sessions" ON chat_sessions
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Users can create their own chat sessions" ON chat_sessions
  FOR INSERT WITH CHECK (auth.uid() = user_id);
```

docker compose logs --tail=200 app | Select-String "MCP servers"
docker compose logs --tail=200 app | Select-String "MCP.*failed"
docker compose logs --tail=200 app | Select-String "provider.probe"

- 排查建议：
  - 校验 `MCP_*` 变量是否匹配平台给出的地址与令牌格式（`Authorization: Bearer <token>`）
  - 确认平台网络可访问相关 SSE/HTTP 端点

## 故障排查

### 常见问题

#### 1. 服务无法启动

**问题：**
```bash
Error: listen EADDRINUSE: address already in use :::3001
```

**解决方案：**
```bash
# 查找占用端口的进程
lsof -i :3001

# 终止进程
kill -9 <PID>
```

#### 2. 数据库连接失败

**问题：**
```bash
Error: connect ECONNREFUSED
```

**解决方案：**
- 检查 Supabase URL 和 Key 是否正确
- 确认数据库服务是否正常运行
- 检查网络连接

#### 3. AI 调用失败

// 实现 JWT 认证
const jwt = require('jsonwebtoken');

function verifyToken(req, res, next) {
  const token = req.headers.authorization;

  if (!token) {
    return res.status(401).json({ error: 'No token provided' });
  }

  jwt.verify(token, SECRET_KEY, (err, decoded) => {
    if (err) {
      return res.status(403).json({ error: 'Invalid token' });
    }
    req.userId = decoded.id;
    next();
  });
}
